# Task Management Backend

## Overview

Production-ready FastAPI backend implementing the Task Management System API contracts.

## Architecture

- **Framework**: FastAPI with Pydantic
- **Database**: SQLAlchemy ORM with SQLite (development)
- **Authentication**: JWT-based with secure password hashing
- **Modules**:
  - Auth: Registration, login, token management
  - Users: Profile and settings management
  - Tasks: Full CRUD with status transitions and history
  - Teams: Team management and membership
  - Comments: Task comments and mentions
  - Notifications: User notifications
  - Reports: Dashboard and analytics

## API Endpoints

### Authentication
- `POST /auth/register` - Register new user
- `POST /auth/login` - Login user
- `POST /auth/refresh` - Refresh access token

### Users
- `GET /users/me` - Get current user profile
- `PUT /users/me` - Update profile
- `GET /users/settings` - Get user settings
- `PUT /users/settings` - Update settings

### Tasks
- `GET /tasks` - List tasks with filtering
- `POST /tasks` - Create task
- `GET /tasks/{id}` - Get task details
- `PUT /tasks/{id}` - Update task
- `POST /tasks/{id}/status` - Update task status
- `POST /tasks/{id}/archive` - Archive task
- `POST /tasks/{id}/restore` - Restore task
- `POST /tasks/{id}/duplicate` - Duplicate task
- `DELETE /tasks/{id}` - Delete task

### Teams
- `GET /teams` - List user teams
- `POST /teams` - Create team
- `GET /teams/{id}` - Get team details
- `PUT /teams/{id}` - Update team
- `POST /teams/{id}/members/{userId}` - Add member
- `DELETE /teams/{id}/members/{userId}` - Remove member

### Comments
- `GET /tasks/{id}/comments` - Get task comments
- `POST /tasks/{id}/comments` - Add comment
- `DELETE /tasks/{id}/comments/{commentId}` - Delete comment

### Notifications
- `GET /notifications` - Get user notifications
- `POST /notifications/{id}/read` - Mark as read
- `DELETE /notifications/{id}` - Delete notification

### Reports
- `GET /reports/dashboard` - Get dashboard metrics
- `GET /reports/workload/me` - Get my workload
- `GET /reports/workload/team/{teamId}` - Get team workload

## Security

- JWT authentication with configurable expiration
- Bcrypt password hashing with configurable policies
- Account lockout after failed login attempts
- SQL injection protection via SQLAlchemy ORM
- CORS configuration per environment
- Input validation with Pydantic

## Configuration

See `src/core/config.py` for all configuration options. Set via environment variables:

- `DATABASE_URL` - Database connection string
- `SECRET_KEY` - JWT secret key
- `DEBUG` - Enable debug mode
- `ENVIRONMENT` - Environment name (development, staging, production)
- `ACCESS_TOKEN_EXPIRE_MINUTES` - Token expiration
- `PASSWORD_*` - Password policy settings
